Huffman Code

     The huffman code is a data/file compression algorithm which has been proven to save typically anywhere from 20% to 90% of memory, according to Introduction to Algorithms, 3rd Edition by Cormen, Leiserson, Rivest, and Stein.
     It operates through assigning codes to characters based on their frequency in the data set instead of giving every character a standard 8 or 16 bit encoding. The idea this algorithm is based around is that not every character in a standard 8 bit encoding scheme will be used in the file and some will be utilized more than others.

Approach

     The goal is to give the characters in the file which appear the most the shortest encoding scheme and the ones that appear the least, the longest.
     We start out by counting the frequency of each character. These frequencies are then sorted in ascending order. From there, we will construct a binary tree of the frequencies:

  1. Pop the first two characters in the list (the first two should have the lowest frequencies).
  2. Create a node with the first node popped as its left child and the second as its right child. This node should record the sum of its children's frequencies as its frequency.
  3. Push this node to the list in sorted order by frequencies.
  4. Repeat steps 1-3 while the array has a length greater than 1.

     Now that the tree is built, the encoding for each letter comes from traversing the tree to that character. While traversing the tree to get to the character, anytime you go left, you add a 0, and anytime you go right you add a 1 to the bit representation for that character.
     What this process has accomplished is that the characters with the higher frequencies will be near the top of the tree which means they will have shorter encodings. Finally, the characters with lower frequencies will be at the bottom of the tree and have the longer bit encodings.
